Command Line: Direct Delete a Forecast from PDL

These command line options let you delete a forecast from PDL. The deletion is sent directly to PDL, without calling Comcat first. So, all necessary information must be given in the command line. Direct deletion never deletes or modifies any existing event-sequence product.

For example, the following command line can be used to direct delete a foreast from the PDL production system. 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --delete --direct=true --code=nc75289501 --eventsource=nc --eventsourcecode=75289501

In this example, the --pdl option specifies that products should be sent to the production PDL system, and the --privateKey option specifies the digital key file that is used to sign products. The --delete option specifies that we want to delete a forecast, and the --direct option specifies that the deletion is to be sent directly to PDL. The --code option gives the "code" that is assigned to the existing product that is to be deleted. The --eventsource option gives the seismic network that reported the mainshock (usually, but not always, a 2-letter identifier). The --eventsourcecode option gives the "code" that the seismic network assigned to the mainshock. Note that the name of the GUI jar file may be different than shown.

Direct delete is intended mainly for use in deleting scenario products. When used for scenario products, you should include the PDL configuration option --type=oaf-scenario. An example of direct deleting a scenario product is: 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --type=oaf-scenario --delete --direct=true --code=nc75289501 --eventsource=nc --eventsourcecode=75289501

When direct deleting a forecast, the following options are required:
    --delete
    --direct=true
    --eventsource=EVENTNETWORK
    --eventsourcecode=EVENTCODE
The following options are optional; if omitted, then the GUI supplies a default value:
    --code=PRODUCTCODE
    --reviewed=ISREVIEWED
The options are described in detail below.

In addition to the options described on this page, you can also use the PDL configuration options. The most often used PDL configuration options are:
    --pdl=PDLDESTINATION
    --privateKey=PRIVATEKEYFILE
    --type=PRODUCTTYPE

Options may appear in the command line in any order.

When you use the command line to delete a forecast, the graphical user interface does not start. After the forecast is deleted, the GUI exits and returns control to the operating system.

 

--delete

The --delete option specifies that you want to delete a forecast from PDL.

 

--direct=true

The --direct=true option specifies that you want to perform a direct delete.

Note: If you omit the --direct option or if you include --direct=false, then the GUI performs a regular deletion instead of a direct deletion.

 

--eventsource=EVENTNETWORK
--eventsourcecode=EVENTCODE

The --eventsource and --eventsourcecode options identify the mainshock. The --eventsource is the seismic network that reported the mainshock (usually, but not always, a 2-letter identifier). The --eventsourcecode is a "code" assigned to the mainshock by the seismic network (an alphanumeric string).

The event ID is obtained by concatenating the values of --eventsource and --eventsourcecode. For example, if you write --eventsource=nc --eventsourcecode=75289501 then the event ID is nc75289501.

 

--code=PRODUCTCODE

The --code option is the "code" that is assigned to the existing product you are trying delete. In order to successfully delete an existing product, the value of --code must match the "code" of the existing product; otherwise, the deletion won't work. Often, but not always, the "code" of the existing product is the mainshock event ID.

If you omit the --code option, then the GUI supplies a default value, which is the event ID derived from the --eventsource and --eventsourcecode options.

 

--reviewed=true
--reviewed=false

The --reviewed option specifies whether the deletion product is marked as being reviewed by an analyst.

If the --reviewed option is omitted, the GUI supplies a default value of true.
