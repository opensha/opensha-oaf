Delete Product

This panel lets you delete the OAF product for the current earthquake from Comcat. Optionally, it can also delete or modify the event-sequence product for the current earthquake.

This function works by sending commands directly to PDL. It does not contact the AAFS servers.

The panel becomes active when you load data into the GUI. (If you load data from a catalog file then you need to check the Call Comcat for mainshock chackbox in order for this panel to be active.) The functions in this panel do not require that you fit parameters or compute a forecast; all that is needed is a mainshock.

 

Event-Sequence Option
Cap Time

Use the drop-down list to select whether you want to delete or modify the event-sequence product for the current earthquake.

  • Delete - If the current earthquake has an event-sequence product, it is deleted.
     
  • Ignore - If the current earthquake has an event-sequence product, it is not deleted or modified.
     
  • Cap End Time - If the current earthquake has an event-sequence product, then the end time of the sequence is changed. This is useful if you are deleting the OAF product because the current earthquake has become a foreshock of a leter, larger earthquake. In this case, you may want to set the end time to be just before the later, larger earthquake. Note that this option can only modify an existing event-sequence product, it cannot create an event-sequence product if one does not already exist.

The text field becomes active if you choose the Cap End Time option. Enter the date and time at which the sequence ends, in UTC. It is recommended that you enter the date and time in the form "yyyy-mm-dd hh:mm", for example, "2025-04-14 17:08". You can also use other time formats.

These controls are disabled if the GUI is configured to not support event-sequence.

 

PDL Delete

Deletes the OAF product of the current earthquake by sending a delete command directly to PDL.

If selected, also deletes or modifies any existing event-sequence product of the current earthquake, by sending commands directly to PDL. It is possible to delete or modify an existing event-sequence product even if the current earthquake does not have an OAF product.

The GUI displays a dialog box for you to comfirm that you want to perform the delete operation.

By default, the PDL destination is Dry Run, which means that the GUI goes through the motions of deleting the OAF product but does not send anything to PDL. In order to send commands to PDL, you must use the GUI command line to specify the PDL destination and private key file.
