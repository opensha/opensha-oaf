Command Line: Deleting a Forecast from PDL

These command line options let you delete a forecast from PDL. You can also delete or modify an existing event-sequence product.

 

Example

For example, the following command line can be used to delete a foreast from the PDL production system, and also delete any existing event-sequence product. 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --delete --eventid=nc75289501

In this example, the --pdl option specifies that products should be sent to the production PDL system, and the --privateKey option specifies the digital key file that is used to sign products. The --delete option specifies that we want to delete a forecast, and the --eventid option gives the event ID of the mainshock. Note that the name of the GUI jar file may be different than shown.

 

Summary

When deleting a forecast, the following options are required:
    --delete
    --eventid=EVENTID
The following options are optional; if omitted, then the GUI supplies a default value:
    --evseq=EVSEQOPTION
    --captime=EVSEQCAPTIME
    --lookbacktime=EVSLOOKBACK
    --reviewed=ISREVIEWED
The following options are obsolete or deprecated; you should avoid using them:
    --code=PRODUCTCODE
    --eventsource=EVENTNETWORK
    --eventsourcecode=EVENTCODE
The options are described in detail below.

In addition to the options described on this page, you can also use the PDL configuration options and the GUI confguration options. The most often used PDL configuration options are:
    --pdl=PDLDESTINATION
    --privateKey=PRIVATEKEYFILE
The most often used GUI configuration option enables or disables support for event-sequence:
    --evseq-config=EVSEQCONFIG

Options may appear in the command line in any order.

When you use the command line to delete a forecast, the graphical user interface does not start. After the forecast is deleted, the GUI exits and returns control to the operating system.

Note that, as an alternative to using the command line, you can also delete a forecast from PDL from within the graphical user interface by opening the Delete Product control panel.

 

Option Details

 

--delete

The --delete option specifies that you want to delete a forecast from PDL.

 

--eventid=EVENTID

The --eventid option specifies the Comcat event ID of the mainshock. For example, --eventid=nc75289501.

 

--evseq=delete
--evseq=ignore
--evseq=cap

The --evseq option specifies what to do if the mainshock has an event-sequence product. It can have the following values:

  • delete - Delete the event-sequence product from PDL.
  • ignore - Do not delete or modify the existing event-sequence product.
  • cap - Change the ending time of the existing event-sequence product. See the description of --captime below.

If the --evseq option is omitted, the default value is delete. If the value is (or defaults to) delete, then this command can delete an existing event-sequence product even if there is no corresponding oaf product.

If there is no existing event-sequence product, this command never creates an event-sequence product even if the value is cap.

If the GUI is configured to not support event-sequence, then an existing event-sequence product is never deleted or modified regardless of the value of --evseq. Note that the --evseq-config option can be used to alter whether or not the GUI is configured to support event-sequence.

 

--captime=EVSEQCAPTIME

If you use --evseq=cap, then you must also include the --captime option. Its value is the new ending time for the event-sequence product. If there is an existing event-sequence product, its ending time is changed to the time given in the --captime option.

On the command line, it is recommended that time be given in ISO-8601 format, for example --captime=2025-04-14T17:08:28Z. You can also use other time formats, but note that if you use a format that contains spaces then you must enclose the time in quotation marks.

Capping is typically used when a mainshock becomes a foreshock of a leter, larger earthquake that occurs nearby. In this situation, you may want to retain the event-sequence product for the original mainshock. but make the sequence end shortly before the later, larger earthquake.

If you don't use --evseq=cap, then the --captime option is not allowed.

 

--lookbacktime=EVSLOOKBACK

The --lookbacktime option specifies how long before a mainshock a sequence begins, measured in days. For example, --lookbacktime=60.0.

if the --lookbacktime option is omitted, the GUI supplies a default value of 30 days.

 

--reviewed=true
--reviewed=false

The --reviewed option specifies whether the deletion product is marked as being reviewed by an analyst.

If the --reviewed option is omitted, the GUI supplies a default value of true.

 

--code=PRODUCTCODE

The --code option is obsolete. If included, it is ignored. It is supported to maintain compatibility with earlier versions of the GUI.

 

--eventsource=EVENTNETWORK
--eventsourcecode=EVENTCODE

The --eventsource and --eventsourcecode options are deprecated. You should avoid using them. They are supported to maintain compatibility with earlier versions of the GUI.

The --eventsource and --eventsourcecode options are an alternative way to identify the mainshock. The --eventsource is the seismic network that reported the mainshock (usually, but not always, a 2-letter identifier). The --eventsourcecode is a "code" assigned to the mainshock by the seismic network (an alphanumeric string). The event ID is obtained by concatenating the values of --eventsource and --eventsourcecode. For example, instead of writing --eventid=nc75289501, you can write --eventsource=nc --eventsourcecode=75289501.
