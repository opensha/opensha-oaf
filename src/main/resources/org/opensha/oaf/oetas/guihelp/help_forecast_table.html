Forecast Table

This window shows the computed forecasts. There can be up to four forecasts:

  • ETAS.
  • Reasenberg-Jones sequence-specific.
  • Reasenberg-Jones generic.
  • Reasenberg-Jones Bayesian.

Use the tabs at the top of the window to select which forecast you want to view. The ETAS forecast, if present, can be either sequence-specific, generic, or Bayesian, depending on the option you selected in the ETAS Options panel when computing the forecast.

The selected forecast is shown in a table at the bottom of the window. The table shows forecasts for four time windows: one day, one week, one month, and one year. Each time window begins at, or shortly after, the end of the loaded earthquake catalog. Normally, this is approximately the current time.

In each time window, forecasts are shown for several magnitude ranges. By default, these are for aftershocks with magnitude at least 3.0, 4.0, 5.0, 6.0, 7.0, and the mainshock magnitude.

In each magnitude range, the table shows the expected number of aftershock, given as the lower and upper limits of the 95% confidence interval, plus the median. The table also shows the probability of one or more aftershocks, given as a percentage.

In the center of the window is a set of controls that you can use to perform the following functions:

  • Save the forecast to a file.
  • Send the forecast to PDL.
  • Set the advisory duration.
  • Select the OAF product template.
  • Specify the time at which the next forecast will be issued.

The controls are described in detail below. Note that there are additional controls in the Analyst Parameters panel that you can use to perform the following additional functions:

  • Enter injectable text.
  • Enter an analyst remark.
  • Set options for the event-sequence product.

 

Export Forecast to File

Lets you save the forecast to a file, in the form of a forecast_data.json file. When you click this button, a file chooser opens to let you enter the filename.

The saved file contains all the forecasts shown in this window, plus the advisory duration, the template, the time of next forecast, the mainshock information, the aftershock search region, the magnitude of completeness parameters, all the forecast parameters, the event-sequence options, the injectable text and analyst remark (if any), and the earthquake catalog. The forecast that is currently displayed in this window is considered to be the selected forecast.

You can later send the saved forecast to PDL using the File Operations panel. If you do, the selected forecast (that is, the forecast that is displayed in this window at the time the file is saved) is the forecast that will appear on the event page.

 

Send Forecast to PDL

Lets you send the forecast to PDL. When you click this button, a dialog box opens asking you to confirm that you want to send the forecast to PDL. The forecast that is currently displayed in this window (that is, the selected forecast) is the forecast that will appear on the event page.

Unlike with earlier versions of the GUI, the forecast will include both forecast.json and forecast_data.json files.

If the event-sequence options (set in the Analyst Parameters panel) specify that an event-sequence product should be sent, then clicking this button also sends the event-sequence product.

By default, the PDL destination is Dry Run, which means that the GUI goes through the motions of sending the OAF and event-sequence products but does not send anything to PDL. In order to send products to PDL, you must use the GUI command line to specify the PDL destination and private key file.

 

Advisory Duration

This drop-down list lets you set the aftershock forecast advisory duration. The possible durations are:

  • 1 Day.
  • 1 Week.
  • 1 Month.
  • 1 Year.

The default advisory duration depends on how long ago the mainshock occurred. It is recommended that the 1 Day duration should not be used.

 

Template

This drop-down list lets you set the OAF product template. The possible templates are:

  • Mainshock.
  • Earthquake of Interest.
  • Swarm.

It is strongly recommended that you leave the template set to Mainshock.

 

Next Forecast
Next Forecast Time

These controls let you specify when the next forecast will be issued. This drop-down list offers the following choices.

  • Not Specified - No information about the next forecast is provided.
     
  • Unknown - Specifies that the next forecast will occur at an unknown time.
     
  • None - Specifies that there will be no further forecasts.
     
  • Set Time - Indicates that the next forecast is scheduled to occur at about the specified time.

If you select Set Time, then the text box is activated. Enter the date and time at which the next forecast is scheduled to occur, in UTC. It is recommended that you enter the date and time in the form "yyyy-mm-dd hh:mm", for example, "2025-04-14 17:08". You can also use other time formats.

 

Export forecast.json

Lets you save the currently-selected forecast to a file, in the form of a forecast.json file. When you click this button, a file chooser opens to let you enter the filename.

The saved file contains only the one forecast that is currently displayed in this window. It also contains the advisory duration, the template, the time of next forecast, and the injectable text (if any) from the Analyst Parameters panel.

You can later send the saved forecast.json file to PDL using the File Operations panel.
