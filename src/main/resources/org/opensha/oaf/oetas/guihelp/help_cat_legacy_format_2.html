Legacy Catalog File Format

Earlier versions of the GUI used a different format for catalog files, which we call the legacy format. The current GUI can read catalog files in the legacy format, but cannot write them.

A legacy catalog file contains a list of earthquakes and nothing more. For each earthquake, the file contains its date and time, latitude, longitude, depth, and magnitude.

We first show an example of a legacy catalog file, and then describe the format in detail.

File Layout

Each line in the file is either an earthquake line or a comment line. A line that starts with # is a comment line. A completely blank line is also considered to be a comment line. All other lines are earthquake lines.

Items appearing within a line can be separated by any number of spaces and/or tabs.

Each earthquake line contains information for one earthquake: year, month, day, hour, minute, second, latitude, longitude, depth, and magnitude, in that order. The year must be given as four dignits; the month, day, hour, minute, and second each must be given as two digits. All times are UTC.

Two special comment lines are used to provide information about the mainshock. The first special comment line begins with the word "Main". The second special comment line, which must appear immediately after the first, contains information for the mainshock: year, month, day, hour, minute, second, latitude, longitude, depth, and magnitude.

All other comment lines are ignored when the catalog file is loaded into the GUI.
