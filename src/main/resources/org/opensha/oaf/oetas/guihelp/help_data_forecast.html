Data Source Options: Published Forecast

This panel can display a list of all published forecasts for the mainshock, and select one to load into the GUI. Only forecasts that include a download file (forecast_data.json) are considered. The mainshock event ID is specified using the USGS Event ID field on the main panel.

The GUI loads the earthquake catalog that is contained within the download file. Also, forecast parameters in the GUI are initialized to the parameter values in the download file that were used to create the published forecast.

 

Populate Forecast List

Click this button to query Comcat and obtain a list of published forecasts that include a download file. The list is loaded into the Available Forecasts drop-down so that you can make a selection.

Note that you don't have to click the button if you just want to use the most recent forecast.

 

Include superseded

Check the box if you want to retrieve a list of all forecasts for the mainshock.

Uncheck the box if you only want to retrieve the most recent forecast.

 

Available Forecasts

If you clicked the Populate button, this drop-down lists all the published forecasts that have a download file. Each forecast is identified by the date and time it was issued. Forecasts are listed in reverse chronologial order. You should select the desired forecast from the list.

If you did not click the Populate button, this drop-down displays Use current forecast, which tells the GUI to load the most recent forecast.

 

Data Start Time
Data End Time

These are read-only fields. When a published forecast is loaded, these fields are set to the start and end times of the earthquake catalog, expressed as days since the mainshock. The start time can be negative, to indicate that the catalog includes foreshocks which occur before the mainshock.
