Analyst Parameters

This panel lets you send a variety of analyst options to the servers. The servers remember these options and use them in future forecasts. You can:

  • Turn forecasts on or off.
  • Select how long the servers should produce forecasts.
  • Enter injectable text or an analyst remark.
  • Select whether the servers generate event-sequence products.
  • Select whether the servers should use custom aftershock parameters.

The panel becomes active when you load data into the GUI. (If you load data from a catalog file then you need to check the Call Comcat for mainshock checkbox in order for this panel to be active.)

Except for using custom aftershock parameters, the functions in this panel do not require that you fit parameters or compute a forecast; all that is needed is a mainshock. If you want to use custom aftershock parameters, then you do need to fit parameters and compute a forecast.

 

Automatic Forecasts

This drop-down list tells the servers whether or not they should generate forecasts for the current earthquake.

  • Normal - Let the servers decide whether or not to generate forecasts using the standard algorithm. The servers generate forecasts automatically if the earthquake is in a supported region, the earthquake's magnitude is above the magnitude threshold for the region, and the earthquake does not appear to be a foreshock or aftershock of a lerger earthquake.
     
  • Enable - Tells the servers to generate forecasts for the current earthquake.
     
  • Disable - Tells the servers to not generate forecasts for the current earthquake.

 

Forecast Duration

Controls how long the automatic system keeps issuing forecasts, in days since the mainshock. The default value of 365 causes the system to issue forecasts for one year after the mainshock. You can enter values up to 3650, which is ten years.

 

Injectable Text

Opens a dialog box for you to enter injectable text. The injectable text appears on the event page for the current earthquake. The injectable text cannot contain hyperlinks or any formatting.

 

Analyst Remark

Opens a dialog box for you to enter an analyst remark. The analyst remark does not appear on the event page for the current earthquake, but it can be viewed by opening the download file (forecast_data.json). The analyst remark cannot contain any formatting. It can contain Internet addresses but they should refer only to highly-reliable pages.

Caution: Although the analyst remark does not appear on the event page, it is visible to the public.

 

Event-Sequence Option
Event Sequence Lookback

The drop-down list tells the servers whether or not they should generate event-sequence products for the current earthquake.

  • Auto - Let the servers decide whether or not to generate event-sequence products using the standard algorithm. By default, the servers generate an event-sequence product whenever they generate a forecast; modify or delete the event-sequence product if the forecast is deleted (for example, if the current earthquake becomes a foreshock of a later, larger earthquake); and let the event-sequence product remain in Comcat after the last forecast is removed.
     
  • Update - Tells the servers to generate event-sequence products for the current earthquake. Currently, this is essentially the same thing as using the standard algorithm.
     
  • Ignore - Tells the servers to not generate, modify, or delete event-sequence products for the current earthquake.
     
  • Delete - Tells the servers to not generate event-sequence products for the current earthquake, and to delete any event-sequence product that already exists.

The text field tells the servers when an earthquake sequence begins, measured in days before the mainshock. The default value is 30 days. Note: This value is used only for generating the event-sequence product, and is used on the event page to display foreshocks. It has no effect on aftershock forecasts.

These controls are disabled if the GUI is configured to not support event-sequence.

 

Use Custom Parameters

Check this box to send custom aftershock forecast parameters to the servers. The servers remember the custom parameters and use them for future forecasts. If the box is not checked, then the servers use the default parameters for the region in which the current earthquakes is located.

The custom parameters include the aftershock search region, magnitude of completeness parameters, b-value, fitting insets, forecast magnitude bins, Reasenberg-Jones parameter ranges, ETAS enable option, ETAS parameter ranges, ETAS model (Bayesian, sequence-specific, generic), and ETAS Bayesian prior.

Note that this checkbox does not become active until you fit forecast parameters and compute a forecast.

 

Export Analyst Options

Lets you save the analyst options to a JSON file. When you click this button, a file chooser opens to let you enter the filename.

You can later send the saved analyst options to the servers using the File Operations panel.

 

Send Analyst Options

Lets you send the analyst options to the servers. When you click this button, a dialog box opens asking you to confirm that you want to send analyst options to the servers.

The GUI comes pre-configured to communicate with a specific server or pair of servers. If the servers require an encrypted connection, then you must install digital certificate files, and the GUI may prompt you to enter a PIN.

The GUI can also be pre-configured not to communicate with any server. In this case, attempting to send analyst options will produce error messages, because the GUI will attempt to contact a non-existent server.
