Command Line: Sending a forecast.json File to PDL

If you have a file that contains a forecast in forecast.json format, you can use these command line options to send the forecast to PDL. You can also send an event-sequence product.

For example, the following command line can be used to send a foreast to the PDL production system. If the GUI is configured to support event-sequence, then the command line also sends an event-sequence product. 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --update=my_forecast.json --eventid=nc75289501

In this example, the --pdl option specifies that products should be sent to the production PDL system, and the --privateKey option specifies the digital key file that is used to sign products. The --update option specifies that we want to send a forecast, and gives the name of the file that contains the forecast.json. The --eventid option gives the event ID of the mainshock. Note that the name of the GUI jar file may be different than shown.

When sending a forecast.json, the following options are required:
    --update=JSONFILENAME
    --eventid=EVENTID
The following options are optional; if omitted, then the GUI supplies a default value:
    --evseq=EVSEQOPTION
    --lookbacktime=EVSLOOKBACK
    --reviewed=ISREVIEWED
The following options are obsolete or deprecated; you should avoid using them:
    --code=PRODUCTCODE
    --eventsource=EVENTNETWORK
    --eventsourcecode=EVENTCODE
The options are described in detail below.

In addition to the options described on this page, you can also use the PDL configuration options and the GUI confguration options. The most often used PDL configuration options are:
    --pdl=PDLDESTINATION
    --privateKey=PRIVATEKEYFILE
The most often used GUI configuration option enables or disables support for event-sequence:
    --evseq-config=EVSEQCONFIG

Options may appear in the command line in any order.

When you use the command line to send a forecast, the graphical user interface does not start. After the forecast is sent, the GUI exits and returns control to the operating system.

Note that, as an alternative to using the command line, you can also send a forecast.json file to PDL from within the graphical user interface by opening the File Operations control panel.

 

--update=JSONFILENAME

The --update option specifies that you want to send a forecast to PDL, and also gives the name of the file that contains the forecast in forecast.json format.

 

--eventid=EVENTID

The --eventid option specifies the Comcat event ID of the mainshock. For example, --eventid=nc75289501.

 

--evseq=update
--evseq=delete
--evseq=ignore

The --evseq option specifies whether to send an event-sequence product. It can have the following values:

  • update - Send an event-sequence product to PDL. The aftershock search region in the forecast.json is used to create the event-sequence product.
  • delete - Delete any existing event-sequence product from PDL.
  • ignore - Do not send an event-sequence product, and do not delete any existing event-sequence product.

If the --evseq option is omitted, the default value is update.

If the GUI is configured to not support event-sequence, then an event-sequence product is never sent, and an existing event-sequence product is never deleted, regardless of the value of --evseq. Note that the --evseq-config option can be used to alter whether or not the GUI is configured to support event-sequence.

 

--lookbacktime=EVSLOOKBACK

The --lookbacktime option specifies how long before a mainshock a sequence begins, measured in days. For example, --lookbacktime=60.0.

if the --lookbacktime option is omitted, the GUI supplies a default value of 30 days.

 

--reviewed=true
--reviewed=false

The --reviewed option specifies whether the forecast is marked as being reviewed by an analyst. The review status is shown on the event page.

If the --reviewed option is omitted, the GUI supplies a default value of true.

 

--code=PRODUCTCODE

The --code option is obsolete. If included, it is ignored. It is supported to maintain compatibility with earlier versions of the GUI.

 

--eventsource=EVENTNETWORK
--eventsourcecode=EVENTCODE

The --eventsource and --eventsourcecode options are deprecated. You should avoid using them. They are supported to maintain compatibility with earlier versions of the GUI.

The --eventsource and --eventsourcecode options are an alternative way to identify the mainshock. The --eventsource is the seismic network that reported the mainshock (usually, but not always, a 2-letter identifier). The --eventsourcecode is a "code" assigned to the mainshock by the seismic network (an alphanumeric string). The event ID is obtained by concatenating the values of --eventsource and --eventsourcecode. For example, instead of writing --eventid=nc75289501, you can write --eventsource=nc --eventsourcecode=75289501.
