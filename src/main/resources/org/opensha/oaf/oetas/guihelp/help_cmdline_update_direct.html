Command Line: Direct Send a forecast.json File to PDL

If you have a file that contains a forecast in forecast.json format, you can use these command line options to send the forecast to PDL. The forecast is sent directly to PDL, without calling Comcat first. So, all necessary information must be given in the command line. Direct send never sends or deletes an event-sequence product.

 

Example

For example, the following command line can be used to send a foreast to the PDL production system. If the GUI is configured to support event-sequence, then the command line also sends an event-sequence product. 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --update=my_forecast.json --direct=true --eventid=nc75289501

In this example, the --pdl option specifies that products should be sent to the production PDL system, and the --privateKey option specifies the digital key file that is used to sign products. The --update option specifies that we want to send a forecast, and gives the name of the file that contains the forecast.json. The --eventid option gives the event ID of the mainshock. Note that the name of the GUI jar file may be different than shown.

Direct send is intended mainly for use in sending scenario products. When used for scenario products, you should include the PDL configuration option --type=oaf-scenario. An example of direct sending a scenario product is: 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --type=oaf-scenario --update=my_forecast.json --direct=true --eventid=nc75289501

 

Summary

When direct sending a forecast.json, the following options are required:
    --update=JSONFILENAME
    --direct=true
    --eventsource=EVENTNETWORK
    --eventsourcecode=EVENTCODE
The following options are optional; if omitted, then the GUI supplies a default value:
    --code=PRODUCTCODE
    --reviewed=ISREVIEWED
The options are described in detail below.

In addition to the options described on this page, you can also use the PDL configuration options. The most often used PDL configuration options are:
    --pdl=PDLDESTINATION
    --privateKey=PRIVATEKEYFILE
    --type=PRODUCTTYPE

Options may appear in the command line in any order.

When you use the command line to send a forecast, the graphical user interface does not start. After the forecast is sent, the GUI exits and returns control to the operating system.

 

Option Details

 

--update=JSONFILENAME

The --update option specifies that you want to send a forecast to PDL, and also gives the name of the file that contains the forecast in forecast.json format.

 

--direct=true

The --direct=true option specifies that you want to perform a direct send.

Note: If you omit the --direct option or if you include --direct=false, then the GUI performs a regular send instead of a direct send.

 

--eventsource=EVENTNETWORK
--eventsourcecode=EVENTCODE

The --eventsource and --eventsourcecode options identify the mainshock. The --eventsource is the seismic network that reported the mainshock (usually, but not always, a 2-letter identifier). The --eventsourcecode is a "code" assigned to the mainshock by the seismic network (an alphanumeric string).

The event ID is obtained by concatenating the values of --eventsource and --eventsourcecode. For example, if you write --eventsource=nc --eventsourcecode=75289501 then the event ID is nc75289501.

 

--code=PRODUCTCODE

The --code option is the "code" that is assigned to the product you are sending. In order to successfully replace an existing product, the value of --code must match the "code" of the existing product; otherwise, the existing product and the product you are sending will both be retained in Comcat. Often, but not always, the "code" assigned to a product is the mainshock event ID.

If you omit the --code option, then the GUI supplies a default value, which is the event ID derived from the --eventsource and --eventsourcecode options.

 

--reviewed=true
--reviewed=false

The --reviewed option specifies whether the forecast is marked as being reviewed by an analyst. The review status is shown on the event page.

If the --reviewed option is omitted, the GUI supplies a default value of true.
