Command Line: GUI Configuration

These command line options are used to configure the GUI. For example, the following command line can be used to enable ETAS forecasts and event-sequence products within the GUI. 

java -jar AftershockGUI-2026_01_06.jar --etas-config=enable --evseq-config=enable

In this example, the --etas-config option specifies that ETAS forecasts should be generated, and the --evseq-config option specifies that event-sequence products should be used. Note that the name of the GUI jar file may be different than shown.

GUI configuration options are described in detail below. All GUI configuration options are optional; if omitted, then the GUI supplies defaults.

Options may appear in the command line in any order.

You can look at the information window to see the GUI configuration options that are currently in effect.

 

--etas-config=enable
--etas-config=disable

The --etas-config option specifies whether the GUI should fit ETAS parameters and generate ETAS forecasts. It can have the following values:

  • enable - Fit ETAS parameters and generate ETAS forecasts.
  • disable - Do not fit ETAS parameters or generate ETAS forecasts.

Note that if the OAF server does not support ETAS, you cannot make the server generate ETAS forecasts by enabling ETAS in the GUI and then sending forecast parameters to the server.

if the --etas-config option is omitted, the GUI supplies a pre-programmed default.

 

--evseq-config=enable
--evseq-config=disable

The --evseq-config option specifies whether the GUI can generate and send event-sequence products. It can have the following values:

  • enable - The GUI can generate and send event-sequence products.
  • disable - The GUI cannot generate and send event-sequence products.

Note that if the OAF server does not support event-sequence products, you cannot make the server generate event-sequence products by enabling event-sequence in the GUI and then sending parameters to the server.

if the --evseq-config option is omitted, the GUI supplies a pre-programmed default.

 

debug

The debug option tells the GUI to run in debug mode. In debug mode, the GUI writes messages to the console that trace the GUI's operation.

If the debug option is included, it must be the first option on the command line. (This is an exception to the general rule that options may appear in any order.)

It is strongly recommended that the debug option not be used when performing production work.
