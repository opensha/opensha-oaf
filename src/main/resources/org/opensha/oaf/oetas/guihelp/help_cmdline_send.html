Command Line: Sending a Forecast to PDL

If you have a file that contains a saved forecast (in forecast_data.json format), you can use these command line options to send the forecast to PDL. The forecast will appear on the event page just like a forecast produced by the OAF servers, with both forecast.json and forecast_data.json files available for download.

An event-sequence product will be sent if the event-sequence options contained within the forecast_data.json file indicate that an event-sequence product should be sent, and if the GUI is configured to support event-sequence.

 

Example

For example, the following command line can be used to send a foreast to the PDL production system. If the GUI is configured to support event-sequence, then the command line may also send an event-sequence product. 

java -jar AftershockGUI-2026_01_06.jar --pdl=prod --privateKey=mykeyfile --send=my_forecast_data.json

In this example, the --pdl option specifies that products should be sent to the production PDL system, and the --privateKey option specifies the digital key file that is used to sign products. The --send option specifies that we want to send a forecast, and gives the name of the file that contains the forecast.json. Note that the name of the GUI jar file may be different than shown. Also, note that you don't need to specify the mainshock event ID, because it is contained within the forecast_data.json file.

 

Summary

When sending a forecast, the following option is required:
    --send=JSONFILENAME
The following option is optional; if omitted, then the GUI supplies a default value:
    --reviewed=ISREVIEWED
The following options are obsolete; you should avoid using them:
    --code=PRODUCTCODE
    --eventsource=EVENTNETWORK
    --eventsourcecode=EVENTCODE
The options are described in detail below.

In addition to the options described on this page, you can also use the PDL configuration options and the GUI confguration options. The most often used PDL configuration options are:
    --pdl=PDLDESTINATION
    --privateKey=PRIVATEKEYFILE
The most often used GUI configuration option enables or disables support for event-sequence:
    --evseq-config=EVSEQCONFIG

Options may appear in the command line in any order.

When you use the command line to send a forecast, the graphical user interface does not start. After the forecast is sent, the GUI exits and returns control to the operating system.

Note that, as an alternative to using the command line, you can also send a saved forecast to PDL from within the graphical user interface by opening the File Operations control panel.

 

Option Details

 

--send=JSONFILENAME

The --send option specifies that you want to send a forecast to PDL, and also gives the name of the file that contains the forecast in forecast_data.json format.

 

--reviewed=true
--reviewed=false

The --reviewed option specifies whether the forecast is marked as being reviewed by an analyst. The review status is shown on the event page.

If the --reviewed option is omitted, the GUI supplies a default value of true.

 

--code=PRODUCTCODE

The --code option is obsolete. If included, it is ignored. It is supported to maintain compatibility with earlier versions of the GUI.

 

--eventsource=EVENTNETWORK
--eventsourcecode=EVENTCODE

The --eventsource and --eventsourcecode options are obsolete. If included, they are ignored. They are supported to maintain compatibility with earlier versions of the GUI.
